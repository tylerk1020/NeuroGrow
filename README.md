# NeuroGrow

AI-powered real-time support for caregivers of individuals with neurological disabilities.

I built this because I have a close family member with a neurological disability, and I've seen firsthand how hard it is to know what to do in the middle of a crisis. The existing resources are generic. Nothing accounts for the specific person in front of you. NeuroGrow is my attempt to change that.

NeuroGrow lets caregivers build a detailed behavioral profile for the person they care for, then report active situations and receive personalized, step-by-step intervention strategies generated by GPT-4o, specific to that individual, not generic advice.

**Live demo:** https://neurogrow-rkxl.onrender.com

> Work in progress. This is an open-source MVP and I'm actively looking for contributors, feedback, and people who work in this space.

---

## What it does

- **Individualized profiles** -- capture communication style, sensory sensitivities, known triggers, escalation signs, calming tools, trusted people, medical notes, and more
- **Real-time situation support** -- describe what's happening right now and get ordered, actionable steps tailored to that person
- **Learns over time** -- rate each strategy after a session; the app synthesizes feedback into behavioral patterns that improve future responses
- **Hard constraints** -- only suggests strategies using items you confirm are present; never suggests do-not-do items; factors in hunger urgency, trusted people nearby, and severity baseline

## Running locally

### Prerequisites

- Python 3.10+
- Node.js 18+
- An OpenAI API key

### Backend

```bash
cd "NeuroGrow Python"
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

Create a `.env` file in the root:

```
OPENAI_API_KEY=your_openai_key_here
SECRET_KEY=your_secret_key_here
DATABASE_URL=sqlite:///./neurogrow.db
```

Start the backend:

```bash
uvicorn app.main:app --reload
```

API runs at `http://127.0.0.1:8000`

### Frontend

```bash
cd frontend
npm install
npm start
```

App runs at `http://localhost:3000`

## Deploying to Render

### Backend (Web Service)

| Field | Value |
|---|---|
| Build Command | `pip install -r requirements.txt` |
| Start Command | `uvicorn app.main:app --host 0.0.0.0 --port $PORT` |

Environment variables to set:

| Key | Value |
|---|---|
| `OPENAI_API_KEY` | your OpenAI key |
| `SECRET_KEY` | a long random string |
| `DATABASE_URL` | Internal Database URL from Render PostgreSQL |
| `ALLOWED_ORIGINS` | your frontend URL |

### Frontend (Static Site)

| Field | Value |
|---|---|
| Root Directory | `frontend` |
| Build Command | `npm install && npm run build` |
| Publish Directory | `build` |

Environment variable:

| Key | Value |
|---|---|
| `REACT_APP_API_URL` | your backend URL |

## Vision

The long-term goal for NeuroGrow is a mobile app that works like a personal AI assistant for caregivers -- something you can pull out in the middle of a crisis and speak to, the way you would Siri, and get an immediate, specific response based on everything it knows about the person you care for.

The key word is specific. Not "try deep pressure." Not "remove sensory stimuli." But the exact strategy, in the exact order, for this person, right now -- informed by their profile, their history, what has worked before, and what has not. The more you use it, the better it gets at understanding that one individual.

To get there, the plan is to move away from general-purpose LLMs and train a dedicated model on real caregiver feedback data. A model that has seen thousands of real situations, rated strategies, and behavioral patterns across many individuals -- and learned from all of it. That is the version of NeuroGrow that could actually make a difference at scale.

This MVP is the foundation: building the profile system, the feedback loop, and the data collection infrastructure that makes that future model possible.
